# EEG Attention Analyzer

A machine-learning pipeline that classifies EEG brainwave recordings as **attentive** or **non-attentive** using 16-channel PiEEG data, with an interactive web UI for visualizing predictions.

---

## Project Structure

```
eeg_analyzer/
├── eeg_ml.py            # Train ML models on reference EEG data
├── predict_eeg.py       # Run predictions on test subject data
├── ui_app.py            # Flask web UI server
├── index.html           # Frontend UI
├── start_ui.sh          # Convenience script to start the UI
├── requirements.txt     # Python dependencies
└── data/
    ├── reference_data.csv        # Training data (19-channel EEG)
    ├── reference_model.joblib    # Saved best model (generated)
    ├── sub1_test_data.csv        # Test subject EEG files
    ├── sub2_test_data.csv
    └── sub3_test_data.csv
```

## Setup

### Prerequisites

- Python 3.9 or later
- `pip` package manager

### 1. Clone the repository

```bash
git clone https://github.com/saratkakarla/eeg_analyzer.git
cd eeg_analyzer
```

### 2. Create and activate a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

The key libraries installed are:

| Library | Purpose |
|---|---|
| `numpy`, `pandas` | Data loading and manipulation |
| `scipy` | EEG signal filtering (bandpass) |
| `scikit-learn` | ML classifiers (Random Forest, Gradient Boosting, Neural Network, Extra Trees) |
| `joblib` | Saving and loading trained models |
| `flask` | Web server for the prediction UI |
| `lightgbm`, `catboost` | Additional gradient boosting backends |
| `plotly` | Interactive charts in the UI |

## Step 1 — Train ML Models (`eeg_ml.py`)

This script reads `data/reference_data.csv`, extracts frequency-band features from each 16-channel EEG window, trains one or more classifiers, and saves the best model to `data/reference_model.joblib`.

### Usage

```bash
python eeg_ml.py [model]
```

### Available models

| Argument | Model |
|---|---|
| `rf` | Random Forest |
| `gb` | Gradient Boosting |
| `nn` | Neural Network (MLP) |
| `et` | ExtraTreesClassifier *(default)* |
| `all` | Train all four models |

### Examples

```bash
# Train the default model (ExtraTreesClassifier)
python eeg_ml.py

# Train a specific model
python eeg_ml.py gb        # Gradient Boosting
python eeg_ml.py rf        # Random Forest
python eeg_ml.py nn        # Neural Network

# Train all models and save the best one
python eeg_ml.py all
```

### Output files

After training, the following files are written to the project directory:

| File | Description |
|---|---|
| `data/reference_model.joblib` | Best-performing model (used by the UI) |
| `all_models.joblib` | All trained models (used by `predict_eeg.py`) |
| `data/cleaned_reference_data.csv` | Preprocessed 16-channel data |
| `eeg_models_accuracy.results` | Accuracy/recall report for all trained models |

> **Note:** To use `predict_eeg.py` or the UI you must run `python eeg_ml.py all` at least once so that `all_models.joblib` is generated.

## Step 2 — Predict on Test Data (`predict_eeg.py`)

This script loads all `*_test_data.csv` files from the `data/` directory and runs predictions using every trained model.

### Prerequisites

`all_models.joblib` must exist (generated by `python eeg_ml.py all`).

### Usage

```bash
python predict_eeg.py [model]
```

Run with a specific model (optional — defaults to ExtraTreesClassifier):

```bash
python predict_eeg.py          # Default: ExtraTreesClassifier
python predict_eeg.py et       # ExtraTreesClassifier
python predict_eeg.py gb       # Gradient Boosting
python predict_eeg.py rf       # Random Forest
python predict_eeg.py nn       # Neural Network
```

### What it does

- Discovers all files matching `data/*_test_data.csv`
- For each test file, extracts the same frequency-band features (delta, theta, alpha, beta) used during training
- Runs prediction with all loaded models and prints a per-subject summary
- Saves detailed results to `subject_prediction.results`

### Example output

```
============================================================
Subject: sub1  (sub1_test_data.csv)
============================================================
  [ExtraTreesClassifier  ]  ATTENTIVE         confidence: 87.43%
  [Random Forest         ]  ATTENTIVE         confidence: 84.11%
  [Gradient Boosting     ]  ATTENTIVE         confidence: 82.56%

Results saved to: subject_prediction.results
```

### Adding your own test data

Place a CSV file named `<subject>_test_data.csv` in the `data/` directory. The file must contain the 16 PiEEG channel columns:

```
Fp1, Fp2, F7, F3, F4, F8, C3, C4, P3, P4, T7, T8, P7, P8, O1, O2
```

No labels column is required — the script predicts the class automatically.

## Step 3 — Start the Web UI (`ui_app.py`)

The web UI lets you select a test subject file and a model, then displays the attentive/non-attentive prediction with confidence scores.

### Prerequisites

`all_models.joblib` must exist (run `python eeg_ml.py all` first).

### Start the server

```bash
python ui_app.py
```

Then open your browser and go to:

```
http://localhost:5004
```

### Using the convenience script

A shell script is provided that kills any existing server on port 5004 before starting a fresh one:

```bash
chmod +x start_ui.sh
./start_ui.sh
```

### UI features

- Select a test subject from the dropdown
- Choose a model (ExtraTreesClassifier, Random Forest, Gradient Boosting, Neural Network)
- View attentive/non-attentive prediction with confidence percentage
- See per-window breakdown (attentive vs non-attentive window counts)
- Compare results across all loaded models

## EEG Signal Processing

The pipeline processes raw EEG samples as follows:

1. **Bandpass filter** — 1–40 Hz Butterworth filter removes low-frequency drift and high-frequency noise
2. **Windowing** — data is split into 2-second windows (256 samples at 128 Hz)
3. **Band power features** — for each channel and each window, power is computed in four frequency bands:
   - **Delta** (1–4 Hz) — deep sleep / drowsiness
   - **Theta** (4–8 Hz) — light sleep / relaxed inattention
   - **Alpha** (8–13 Hz) — relaxed wakefulness
   - **Beta** (13–30 Hz) — active thinking / attention
4. **Classification** — the feature vector (16 channels × 4 bands = 64 features) is passed to the trained classifier

## Troubleshooting

**`Error: File not found at data/reference_data.csv`**
Make sure `data/reference_data.csv` exists before running `eeg_ml.py`.

**`Error: Models file not found at all_models.joblib`**
Run `python eeg_ml.py all` first to generate `all_models.joblib`.

**`Error: No *_test_data.csv files found in data/`**
Place at least one `*_test_data.csv` file in the `data/` directory before running `predict_eeg.py`.

**Port 5004 already in use**
Run `./start_ui.sh` which automatically kills any existing process on that port, or manually: `lsof -ti:5004 | xargs kill -9`.
